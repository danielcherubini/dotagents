Pre-delivery verification checklist for research reports.

## Evidence Checklist

- [ ] Every claim is backed by at least one cited source
- [ ] Verified facts are distinguished from inferences and speculation
- [ ] Contradictions between sources are identified and addressed
- [ ] No source was silently discarded — disagreements are explained
- [ ] Cross-referencing was done for important claims (2+ sources)
- [ ] Primary sources were preferred over secondary summaries

## Citation Checklist

- [ ] All code citations use commit SHAs (not branch names)
- [ ] All citations include enough info to locate the source
- [ ] Dates are included for time-sensitive sources
- [ ] Versions are noted for library/framework documentation
- [ ] Timestamps are included for video-specific claims
- [ ] Persistent URLs are used (DOIs, permalinks) where available
- [ ] Paywalled or unavailable sources are marked as such

## Structure Checklist

- [ ] Report has an executive summary
- [ ] Findings are organized by research question (not by source)
- [ ] Each finding has at least one citation
- [ ] Unresolved contradictions have their own section
- [ ] Gaps section lists what remains unknown
- [ ] Report length matches research depth (quick = brief, deep = full)

## Credibility Checklist

- [ ] Key sources were assessed for credibility
- [ ] Higher-credibility sources were weighted appropriately
- [ ] Domain-appropriate hierarchy was applied (not just default)
- [ ] Red flags were checked (author, evidence, currency, independence)
- [ ] Vendor/sponsored content was identified and noted
- [ ] Anonymous or low-credibility sources were cross-referenced

## Gap Checklist

- [ ] All research questions from the plan are addressed (or flagged as gaps)
- [ ] Gaps explain why the information wasn't available
- [ ] Gaps suggest how the user could continue research if needed
- [ ] Stale sources are noted with currency warnings
- [ ] Paywalled sources are noted with workaround suggestions
- [ ] The report is honest about what is unknown
